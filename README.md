## 一、這是什麼？

**便利君（Benrikun，日文「便利屋」之意）** 是一款專為「**大批量、重複性**」辦公室文件作業而生的桌面工具。

日常工作中，我們常遇到這類令人頭痛的場景：

- 一個資料夾裡躺著 50 份 Word，要逐一**轉 PDF**、**算頁數**、**檢查失效連結**……
- 用 Excel 名單**套印**幾百份通知書，一份份手動改名字？
- 十幾份簡報要**合併成一份**，母片、字型還要統一？
- 簡報想轉成**純圖片版**防止被竄改，或擷取文字成 **Markdown** 餵給 AI？
- 出遊回來幾千張相片，想**依拍攝日期自動分資料夾**？

便利君把這些「**選一個目錄 → 按下執行 → 喝杯咖啡**」就能解決。

### ✨ 特色

| 特色 | 說明 |
|------|------|
| 🗂️ **目錄即輸入** | 不挑單檔，整個目錄（含子目錄）批次處理 |
| ⚡ **背景執行不卡頓** | COM 操作全在背景執行緒，UI 隨時可取消 |
| 📊 **即時進度回報** | 每處理一個檔案即時更新表格，雙擊可開啟所在目錄 |
| 🧹 **乾淨收尾** | 嚴格管理 Office COM 生命週期，不殘留殭屍程序 |
| 🎨 **現代化介面** | WPF Fluent 主題，圓角、陰影、深淺色自動適應 |

## 三、各功能詳解

### <span style="color: blue;"> 1. Excel + Word 大批文件產製 </span>

> **用 Excel 名單套印 Word 範本，一次產出大批文件。**

在 Word 範本中以 `${A}`、`${B}`、`${C}`…… 作為占位符（對應 Excel 的 A、B、C 欄）。程式逐列讀取 Excel 資料，將每一列的欄位值取代範本中的對應占位符，每一列各自另存成一份獨立的 Word 文件。

- 占位符規則：`${欄位代號}`，例如 `${A}` 代表 Excel 第 A 欄
- 取代引擎：Word COM `Find/Replace`，支援換行符正確還原
- 輸出：每列一份 Word，DataGrid 即時列出產生檔案（雙擊開啟目錄）

```
┌─ Excel + Word 大批文件產製 ──────────────────────────────┐
│ [▶ 執行]                                                  │
│ ┌─ 設定 ──────────────────────────────────────────────┐  │
│ │  Word 範本   [ 通知書範本.docx           ▾] [📁]     │  │
│ │  Excel 檔案  [ 學員名單.xlsx ▸ 工作表1    ▾] [📁]     │  │

> **掃描目錄下所有 Word 的超連結，自動分類並驗證有效性。**

逐一開啟 Word，分析每一條超連結並依規則分類：

| 類型 | 判斷依據 | 結果 |
|------|----------|------|
| 章節鏈結 | `_Toc` 開頭 | O 正常 |
| 網頁鏈結 | `http` 開頭 | O 正常 |
| 信箱鏈結 | `mailto` | O 正常 |
| 內部鏈結 | 無 Address 有 SubAddress | O 正常 |
| 絕對路徑 | 含 `:\` | ✗ 錯誤 |
| 外部檔案 | 有 Address | 檢查檔案是否存在 |

### <span style="color: blue;"> 5. PowerPoint 大批合併 > PPTX </span>

> **將目錄下所有簡報依檔名排序，合併為單一 PPTX，並可統一字型、剔除重複頁。**

- **合併策略**：採 COM 剪貼簿 + `PasteSourceFormatting`（保留來源格式），讓 PowerPoint 引擎自動處理母片複製、版面對應與 XML ID 重命名，避開脆弱的手動 XML 解析
- **章節保留**：自動重建各來源簡報的章節（Section），重複名稱加上 `(2)`、`(3)` 後綴
- **版面驗證**：合併前後各投影片匯出縮圖，以 MD5 逐張比對，回報可能跑版的頁面
- **剔除重複**（選填）：將投影片匯出 PNG 算 MD5，像素完全相同者只留第一張
- **字型統一**（選填）：可分別指定中文／英文字型，留空則保留原字型

```
┌─ PowerPoint 大批合併 > PPTX ──────────────────────────────┐
│ [▶ 執行]                                                   │
│ ┌─ 設定 ───────────────────────────────────────────────┐  │
│ │  來源目錄  [ D:\簡報素材\           ▾] [📁]            │  │
│ │  ☑ 剔除重複投影片                                     │  │
│ │  中文字型  [ 微軟正黑體     ]（留空保留原有字型）     │  │
│ │  英文字型  [ Calibri        ]（留空保留原有字型）     │  │
│ └──────────────────────────────────────────────────────┘  │
│  進度：合併 → 版面驗證 → 剔除重複 → 字型套用 → 存檔      │
└─────────────────────────────────────────────────────────────┘
```

### <span style="color: blue;"> 7. PowerPoint 大批圖片化 > PPTX </span>

> **將每張投影片轉成高畫質圖片，再組成「純圖片版」簡報——防竄改、跨平台零跑版。**

適合對外發送、上傳投稿時，避免對方因缺字型／版本不同而跑版，也防止內容被隨意修改。

- **匯出解析度**：長邊固定 1920px，依原始比例**動態計算**（16:9 → 1920×1080、4:3 → 1920×1440），永不拉伸變形
- **圖片格式**：PNG（無損）
- **背景填滿法**：圖片以投影片背景（`Background.Fill`）方式填入，使用者無法誤點移動，且 100% 相容純背景執行
- **雙重輸出**（與來源同目錄，整批共用一個時間戳記）：
  - `{原檔名}_Images_{時間戳}.pptx` — 純圖片版簡報
  - `{原檔名}_Images_{時間戳}/` — 依投影片順序編號的 PNG 圖片集

```
┌─ PowerPoint 大批圖片化 > PPTX ────────────────────────────┐
│  選擇目錄 ▸ 每份 PPTX 唯讀開啟 ▸ 逐頁匯出 1920px PNG       │
│           ▸ 新建空白簡報 ▸ 圖片滿版貼為背景 ▸ 另存          │
│                                                            │
│  ┌─ 進度 ─────────────────────────────────────────────┐  │
│  │ 輸出 PPTX (雙擊開啟)         │ 頁數 │ 說明           │  │
│  │ 提案_Images_20260602_..pptx │  18  │ 完成，18 頁    │  │
│  └────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### <span style="color: blue;"> 8. PowerPoint 大批內容擷取 > Markdown </span>

> **把簡報的文字、表格、SmartArt、備忘稿擷取成 Markdown，並抽出圖片／影片媒體——餵 AI、做文件、轉 HTML 都好用。**

- **支援的形狀**：標題、文字框、表格（轉 Markdown pipe 格式）、群組（遞迴展開）、SmartArt（保留層次縮排）、備忘稿、隱藏投影片標記
- **章節感知**：有章節時標題降為 H3，無章節時維持 H2
- **媒體擷取**：透過 OOXML（ZIP）直接解析 `.pptx`，**不經 COM** 抽出原始畫質的圖片與影片，以 MD5 跨頁／跨檔去重後注入合併的 `_merged_xxx.md`
- **智慧過濾**：只擷取「圖片」與「影片」；向量圖（EMF/WMF/SVG）、音訊、其他類型自動略過

```
# 檔案名稱
## 📂 章節名稱
### 第 N 頁：投影片標題
[內文、SmartArt bullet、表格]
![圖片：第 N 頁 - 第 M 個](media/...)
🎬 [影片：第 N 頁 - 第 M 個](media/...)
#### 備忘稿
```

```
┌─ PowerPoint 大批內容擷取 > Markdown ──────────────────────┐
│  ┌─ 進度 ─────────────────────────────────────────────┐  │
│  │ 來源檔案    │ 投影片數 │ 輸出 .md（雙擊開啟目錄）   │  │
│  │ 年報.pptx   │   24     │ D:\簡報\年報.md            │  │
│  │（合併檔案） │ 8 個媒體 │ D:\簡報\_merged_2026...md  │  │

### <span style="color: blue;"> 9. 相片日期分目錄 </span>

> **掃描相片目錄，依「拍攝日期」自動建立子資料夾歸檔。**

- **日期來源**：優先讀取 EXIF `DateTaken`；讀不到（HEIC 未裝 Codec、RAW 無對應 WIC、檔案損毀）則 fallback 至檔案最後修改時間
- **多格式支援**：JPG/PNG/GIF/WebP/TIFF、Apple HEIF（HEIC/HEIF）、各家 RAW（CR2/NEF/ARW/DNG…）
- **子目錄格式可自訂**：`yyyy-MM-dd`、`yyyyMMdd`、`yyyy年MM月dd日`… 或自行輸入 DateTime 格式
- **複製 / 移動**可選
- **聰明分流**：非照片檔案（文件、影片）一律移到 `[archive_日期時間]` 子目錄；系統暫存檔（Thumbs.db、.DS_Store、`._*`、`~$*`）自動略過

```
┌─ 相片日期分目錄 ──────────────────────────────────────────┐
│ [▶ 執行]                                                   │
│ ┌─ 設定 ───────────────────────────────────────────────┐  │
│ │  來源目錄    [ E:\相機匯入\         ▾] [📁]            │  │
│ │  子目錄格式  [ yyyy-MM-dd          ▾]                 │  │
│ │  ◉ 將照片複製到子目錄   ○ 將照片移動到子目錄          │  │
│ │  ℹ 非照片移至 [archive_…]；系統暫存檔自動略過         │  │
│ └──────────────────────────────────────────────────────┘  │
│  ┌─ 進度 ─────────────────────────────────────────────┐  │
│  │ 目的目錄 (雙擊開啟) │ 檔案數 │ 說明                  │  │
│  │ E:\相機匯入\2026-05-30 │  42  │ 照片                │  │
│  │ E:\相機匯入\[archive_…]│   3  │ 其他檔案            │  │
│  └────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```
